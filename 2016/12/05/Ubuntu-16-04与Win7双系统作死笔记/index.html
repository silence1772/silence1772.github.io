Ubuntu 16.04与Win7双系统作死笔记

学习
  • Linux

一 前言

作为计算机专业的学生，玩Linux是必须的，于是就想win7的基础上再装一个Linux系统，组成双系统。Linux就用于工作，Windows就用于休闲。不过Linux这么多发行版，挑哪一个着实困扰了我，最后还是选择了Ubuntu 16.04 LTS。原因有以下几点：首先Ubuntu的图形化界面做得非常不错，对于新手来说比较容易入门，尤其是习惯了Windows的人。其二就是作为个人用户，如果不是打算搞服务器编程方面的，其实没有必要一开始就上centos和Debian，完全可以等到需要时才去装。

简介

Ubuntu（乌班图）是一个以桌面应用为主的Linux操作系统，其名称来自非洲南部祖鲁语或豪萨语的“ubuntu”一词，意思是“人性”、“我的存在是因为大家的存在”，是非洲传统的一种价值观，类似华人社会的“仁爱”思想。Ubuntu基于Debian发行版和GNOME桌面环境，而从11.04版起，Ubuntu发行版放弃了Gnome桌面环境，改为Unity，与Debian的不同在于它每6个月会发布一个新版本。Ubuntu的目标在于为一般用户提供一个最新的、同时又相当稳定的主要由自由软件构建而成的操作系统。Ubuntu具有庞大的社区力量，用户可以方便地从社区获得帮助。

二 安装

1.

首先到Ubuntu官网下载Ubuntu 16.04，这个用迅雷下载会非常快，1G多我用了一分多钟就搞定了。

2.

百度搜索ultraISO下载并安装,插入U盘（容量最好2G以上，并且格式化），打开软件后按下图操作



然后点写入，等待完成。如果速度太慢的话可以考虑 U盘复制速度太慢怎么办？-图文详细版

3.

右键点击我的电脑》管理》磁盘管理,选择一个空间较大的盘,右键选择压缩卷,我为Ubuntu16.04划分了80G的空间(空间大小根据自己的需要自行选择,建议如果只是安装作为练习使用有30G就可以,要是自己有运行的程序或者要处理数据还是建议多分一点空间),设置好容量大小后点压缩即可,保持它的绿色状态，不用给他设置盘符,我们在安装Ubuntu的时候在重新分区。注意:这种分区我们在Win7里面不能直接查看Ubuntu里的分区,但是在Ubuntu我们可以访问放在硬盘上的任何东西,也就是可以直接查看Win7系统的分区

4.

将Ubuntu的启动U盘插在电脑上,重启电脑,进入BIOS,选择U盘启动。我的是华硕主板，开机时狂按F8键就可以进入选择菜单。
选择U盘启动后,进入Ubuntu的安装界面,选择语言》中文(简体),点击安装Ubuntu，不要选那个安装时下载更新，会很慢。在安装类型界然后选择最后一项“其他选项”，这样可以自己手动分区，点击继续。 现在我们看到的是硬盘的分区情况，找到前边有“空闲”二字，我们要做到就是，把空闲的空间给ubuntu划分分区。

点击“空闲”的分区，选择下边的“+”，弹出窗口，上边填写分区空间大小，下边填写要挂载的分区，然后确定。

逻辑分区，200M，起始，Ext4日志文件系统，/boot；（引导分区200M足够）

逻辑分区，4000M，起始，交换空间，无挂载点；（交换分区swap，一般不大于物理内存）

逻辑分区，20000M，起始，Ext4日志文件系统，/；（系统分区”/”或称作”/root”装系统和软件，15G以上足够）

逻辑分区，剩余空间数，起始，Ext4日志文件系统，/home；（home分区存放个人文档）

接下来这一步非常重要！！！分区设置好后，查看/boot分区的编号，然后在下边的“安装启动引导区的设备”下拉框中选择/boot分区的

编号，点击安装。一直到你安装成功，当然中途需要你设置一个用户名和密码的（这个就不说了吧）

安装完成后需要重新启动（这个时候默认是在Ubuntu启动模式下进入的–》选择windows7系统启动） 还有键盘设置选美式键盘。在安装过程中还会显示在下载语言包，旁边有个skip按钮，点多几下可以跳过，因为实在是太慢了。

5.

1.下载EasyBCD，此软件用于在启动电脑的时候选择要进入的系统（自行百度搜索安装）

2.打开easyBCD，选择add new entry, 选择Linux/BSD, name这一行随便填写，只是系统名词，写ubuntu吧，Device这一行选择刚刚我们创建的200MB的那个”/boot“分区,前边有linux标记的。（其他的不要动）

开启系统

做完这些重启系统后，系统会将win7系统和ubuntu 16.04系统都列出来，你可以选择系统进入了。

这样启动的好处（windows 不会受到Ubuntu的影响）

如果说没有最后这一步，没有任何问题，但是你要是启动windows7把Ubuntu系统的分区删除，那么就启动不了系统了

（如果你真的遇到这样的问题了，不要着急，直接用老毛桃U盘或者大白菜制作好的U盘启动，直接启动引导修复就OK了）

三 作死开始

1

进入Ubuntu后系统提示有可用更新，我把系统各个地方熟悉之后就打开了软件更新器，下载安装了更新，里面有一个系统的核心组件更新，就是把16.04升级为16.04.1，结果升级好了之后重启就打不开了，只有一个黑乎乎的界面和左上角不断闪烁的光标。在百度了许多次之后，我重新回到WIN7下把EasyBCD中原来的linux条目删掉再添加就可以了。

2

重新启动后可以正常启动进入，然后作死把旧的内核还有不用的软件包删掉了，接着几次都可以正常开关机使用，然而第二天进入时却显示The system is running in low-graphics mode

不管怎样点都没用，查百度说是驱动问题，用了好几个办法都没用。心情很糟糕，就想着破罐子破摔再安装一次算了，于是就再插入U盘重装，结果在选择安装类型时以为理所当然的选了第一个 覆盖前一个Ubuntu系统(其实是要选那个其他选项的)，安装完后发现竟然只安装在了/目录下，意思就是之前分的4个区只用到了20G的根目录，其他3个区都没用到，只有20G怎么够用啊，于是我选择了再一次重装Ubuntu，这次安装挂载在了之前分的几个区上了，重启，竟然两个系统都无法进入了，只出现了error: file ‘/boot/grub/i386-pc/normal.mod’ not found.
Entering rescue mode…
grub rescue>

其实是进入了UBUNTU的救援模式，因为第二次安装时选择的是覆盖安装，相当于自动安装，引导程序被安装在sda11(我划分的“/”区)上的/boot/grub，但第三次我却选择安装在sda8上的/grub(sda8就是我划分的/boot)，而且第二次安装时grub(ubuntu的引导)被写在了MBR上覆盖掉了之前的Windows的引导，因此开机时他先启动的是Ubuntu的grub，而grub因为没有更新所以它加载的还是第二次安装时的在sda11上的引导程序，而此时里面的内容早以没有了，所以就出现了Entering rescue mode…进入救援模式。
解决过程主要参考了Grub Rescue> 修复方法
以及开机启动遇到grub rescue，无法启动系统怎么办

我的具体解决方法是：
1.我知道我的引导是在sda8上，并且我是单独分区，所以在路径前我不用加/boot
2.依次输入
set root=(hd0,msdos2)/grub
set prefix=(hd0,msdos2)/grub
insmod normal
normal
然后就能正常进入系统了，但是到这里还没完，还有进入Ubuntu的终端更新GRUB菜单
3.在终端输入
sudo update-grub
sudo grub-install /dev/sda
这样就OK了，可以正常进入系统了，不过默认的还是Ubuntu的grub。。。
