hexo博客提交搜索引擎收录及统计(百度+google)

学习

提交搜索引擎

博客搭好之后虽然百度上搜site:silence1772.cn能搜到自己网站，可是搜文章的时候还是搜不到，所以就想自己提交到搜索引擎上，百度方法后大概是这样完成的：
1.先登录百度站长平台和谷歌serach console验证主站。
这里验证主站有三种方式，先进行百度的验证
百度站长平台为未使用百度统计的站点提供三种验证方式：文件验证、html标签验证、CNAME验证。
　　1.文件验证：您需要下载验证文件，将文件上传至您的服务器，放置于域名根目录下。
　　2.html标签验证：将html标签添加至网站首页html代码的标签与标签之间。
　　3.CNAME验证：您需要登录域名提供商或托管服务提供商的网站，添加新的DNS记录。
验证完成后，我们将会认为您是网站的拥有者。为使您的网站一直保持验证通过的状态，请保留验证的文件、html标签或CNAME记录，我们会去定期检查验证记录。

开始我选择第一种文件验证，下载验证文件后放在博客根目录的source下，生成g 部署d 后点完成验证却出现了 “未知原因错误：301”，搜了下发现是这个html文件在执行hexo g 命令时被加上了博客的模板变成了静态网页，而百度要的却只是html文件里的一行验证字符，于是失败。
然后自然是选择第二种，因为我用的主题配置文件里有关于百度谷歌站长验证的配置，所以我就直接将百度给出的一行html标签中content=”….”中的内容写进配置文件就行了，然后编译 部署。这里有一个坑就是我的博客根目录配置文件_config.yml里写的url是没有www的silence1772.cn，但是因为在百度站长里添加主站的时候它提示推荐使用WWW，所以我就填了www.silence1772.cn，结果就是在验证html标签的时候一直不通过，试了n次，后来我就只能使用第三种CNAME的方式在腾讯云解析那边解析到zz.baidu.com，成功了，再接着在谷歌search console那边用html标签的方式验证网站竟然通过了，马上就意识到是因为这个域名填写的原因，返回百度这边修改成silence1772.cn后就成功了。

2.接着就是提交站点地图

站点地图是一种文件，您可以通过该文件列出您网站上的网页，从而将您网站内容的组织架构告知Google和其他搜索引擎。Googlebot等搜索引擎网页抓取工具会读取此文件，以便更加智能地抓取您的网站。

先在根目录下执行

npm install hexo-generator-sitemap –save
npm install hexo-generator-baidu-sitemap –save
然后hexo g -d后访问
http://silence1772.cn/sitemap.xml
http://silence1772.cn/baidusitemap.xml
出现内容后就是成功了。
然后就分别在百度谷歌提交对应的站点地图就可以了
